# Exercise 2: Importing a Data Shortcut Project into Vault

In this exercise, you will import a data shortcut project into Vault, creating a new Vault project in the process.

Project import is more than just the reverse of export. You can use this operation to import a data shortcut project into Vault, or to import several drawings that are not part of a formal project. In the import process, any data shortcut references are converted into Vault references.

Note:

Before you start this exercise, you must have created the data shortcuts project that is described in the [Using Data Shortcuts tutorial](GUID-68AC7599-F635-4391-9F9A-6E9EF96F9ED2.htm "This tutorial demonstrates how to create a data shortcuts project, create data shortcuts from objects in one drawing, and then import the data shortcuts into another drawing.").

This exercise continues from [Exercise 1: Exporting a Vault Project](GUID-CBE305B8-DDA1-4A24-AF75-913DAB1A146D.htm "In this exercise, you will export a Vault project into a portable ZIP file.").

Import a project

1.  In Toolspace, on the Prospector tab, right-click Projects node ![](../images/ac.menuaro.gif)Import From![](../images/ac.menuaro.gif)Folder.
    
    Notice that you can also import from a ZIP file. Typically, you would use this option to import a project that had been exported from Vault.
    
2.  In the Browse For Folder dialog box, navigate to the [My Civil 3D Tutorial Data folder](GUID-DECD2305-4906-4329-A973-CFC9B625B4CD.htm#GUID-DECD2305-4906-4329-A973-CFC9B625B4CD__WS1A9193826455F5FF1848157110156D8271-7832). Select **Tutorial Data Shortcuts Project**. Click OK.
    
    You see a progress bar and several command-line messages about the operation. When the project is imported, it appears in the Projects collection on the Prospector tab, and a copy is created in your working folder Tutorial Data Shortcuts Project is now a Vault project.
